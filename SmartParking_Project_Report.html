Chapter 1: Introduction

1.1 Existing System and Need for System

In modern urban environments, parking management has become one of the most pressing challenges faced by city dwellers, shopping malls, hospitals, and educational institutions. The conventional approach to parking relies entirely on manual operations — security guards directing vehicles, physical token systems, and paper-based records — all of which are inherently inefficient, error-prone, and unable to scale with rising vehicle populations.

Traditional parking systems suffer from numerous shortcomings. Drivers must physically explore an entire parking facility looking for a vacant spot, wasting significant fuel and time. There is no centralized visibility into slot availability, no digital record of active bookings, and no mechanism for prior reservation. Disputes over occupied slots are common, and revenue leakage due to untracked entries is significant.

Furthermore, the COVID-19 pandemic accelerated the demand for contactless, digital-first solutions across all sectors. Parking is no exception. The need for a system that allows users to browse available parking spots, make reservations in advance, receive real-time email confirmations, and manage their bookings entirely from a web browser is more urgent than ever.

The SmartParking Management System was conceived to directly address all these pain points. It replaces fragmented, manual workflows with a fully integrated, web-based platform that connects three key actors: Parking Hosts (who list and manage their spots), Users (who search, book, and cancel reservations), and Administrators (who oversee platform-wide operations). Every significant action — registration, login, booking, and cancellation — is accompanied by an automated email notification, ensuring transparency and trust at every step.

1.2 Scope of Work

The scope of the SmartParking project encompasses the design, development, testing, and deployment of a full-stack web application for intelligent parking spot management. The system is built using ASP.NET Core 8.0 for the backend REST API, a vanilla HTML/CSS/JavaScript frontend, and MySQL as the persistent relational database.

The key functional areas covered within this project are:

  • User Authentication: Secure registration and login using JWT (JSON Web Tokens) with role-based access control (User, Host, Admin).
  • Parking Spot Management: Hosts can list, update, and remove parking spots with details such as name, address, GPS coordinates, price per hour, and total capacity.
  • Booking System: Registered users can search for available spots, select time slots, add their vehicles, and confirm reservations. The system enforces conflict detection to prevent double-booking.
  • Email Notification Service: An integrated SMTP email service using Gmail dispatches automated confirmations for registration, login alerts, booking confirmation, cancellation, and password reset.
  • Forgot Password Flow: A secure, token-based password reset mechanism generates a 6-digit OTP with a 15-minute expiry, delivered via email, allowing users to reset their passwords safely.
  • Vehicle Management: Users can register their vehicles and associate them with bookings.
  • Admin Dashboard: An administrative interface for platform management.
  • Camera Integration: A camera controller supports entry/exit image capture endpoints for advanced tracking.

The scope explicitly excludes payment gateway integration, mobile application development, and IoT-based physical barrier automation, which are planned for future enhancements.

Chapter 2: Proposed System

2.1 Proposed System

The SmartParking Management System is proposed as a comprehensive, cloud-ready, full-stack web application designed to digitalize and automate all aspects of parking spot management. The system operates on a client-server architecture where a RESTful API backend (ASP.NET Core) serves a Single Page Application (SPA) frontend (HTML/JS) through static file serving.

The system is structured around three distinct roles:

  • Admin: Has unrestricted access to the entire platform. Can manage users, monitor all bookings, and oversee parking spot listings.
  • Host: Can register their parking facilities, list available spots with pricing and capacity, and view bookings made at their locations.
  • User: Can search for available parking spots, make time-based reservations, register vehicles, cancel bookings, and reset their password securely.

The proposed system replaces all manual record-keeping with a persistent MySQL database managed through Entity Framework Core. All user sessions are secured via JWT tokens issued at login, with token expiry of 24 hours. All critical events trigger automated email notifications through the integrated Gmail SMTP service, providing an audit trail and improving user trust.

The system architecture follows the MVC (Model-View-Controller) pattern on the backend. The frontend communicates exclusively through RESTful JSON APIs, making the system easily extensible to mobile applications in the future.

2.2 Objectives of System

The primary objectives of the SmartParking Management System are as follows:

  1. Digitalize Parking Management: Eliminate manual, paper-based processes by providing a fully digital platform for listing, searching, booking, and managing parking spots.
  2. Prevent Double Booking: Implement robust conflict detection that checks for overlapping reservations before confirming any new booking, ensuring data integrity.
  3. Secure Authentication: Provide a robust, token-based authentication system with role-based access control to protect sensitive data and restrict unauthorized actions.
  4. Real-time Email Notifications: Automatically notify users via email at every key interaction — account registration, login, booking confirmation, booking cancellation, and password reset.
  5. Secure Password Recovery: Implement a safe, time-limited (15-minute OTP) forgot password flow that delivers a 6-digit confirmation code to the user's registered email.
  6. Vehicle Registration: Allow users to associate vehicles with their profiles, enabling vehicle-specific parking reservations.
  7. Multi-role Dashboard: Provide role-appropriate dashboards for Users (booking history), Hosts (spot and revenue management), and Admins (platform oversight).
  8. Scalability: Build on a robust, industry-standard technology stack (ASP.NET Core + MySQL + EF Core) that can scale to handle thousands of concurrent users.

4.3 Project Code

AuthController – Register Endpoint

[HttpPost("register")]
public async Task<IActionResult> Register(RegisterDto dto)
{
    if (await _context.Users.AnyAsync(u => u.Email == dto.Email))
        return BadRequest("Email already exists.");

    if (!Regex.IsMatch(dto.Password,
        @"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[^\da-zA-Z]).{8,}$"))
        return BadRequest("Password does not meet complexity requirements.");

    var user = new User {
        Name = dto.Name, Email = dto.Email,
        PhoneNumber = dto.PhoneNumber,
        PasswordHash = dto.Password, Role = dto.Role
    };
    _context.Users.Add(user);
    await _context.SaveChangesAsync();

    _ = _emailService.SendEmailAsync(user.Email, "Welcome to SmartParking!",
        $"<h1>Welcome, {user.Name}!</h1><p>Your account is now active.</p>");

    return Ok("User registered successfully.");
}

AuthController – Forgot Password Endpoint

[HttpPost("forgot-password")]
public async Task<IActionResult> ForgotPassword(ForgotPasswordDto dto)
{
    var user = await _context.Users
        .FirstOrDefaultAsync(u => u.Email == dto.Email);
    if (user == null) return NotFound("Email not found.");

    var code = new Random().Next(100000, 999999).ToString();
    user.ResetPasswordToken = code;
    user.ResetPasswordExpiry = DateTime.Now.AddMinutes(15);
    await _context.SaveChangesAsync();

    _ = _emailService.SendEmailAsync(user.Email,
        "SmartParking Password Reset Code",
        $"<h2>Your code: <strong>{code}</strong></h2>" +
        "<p>Valid for 15 minutes.</p>");

    return Ok("Password reset code sent to your email.");
}

BookingController – Create Booking

[Authorize]
[HttpPost]
public async Task<IActionResult> CreateBooking(CreateBookingDto dto)
{
    var spot = await _context.ParkingSpots.FindAsync(dto.ParkingSpotId);
    if (spot == null) return BadRequest("Spot not found.");

    var overlap = await _context.Bookings
        .Where(b => b.ParkingSpotId == dto.ParkingSpotId
                 && b.Status != "Cancelled"
                 && dto.StartTime < b.EndTime
                 && b.StartTime < dto.EndTime)
        .FirstOrDefaultAsync();

    if (overlap != null)
        return BadRequest("Spot already booked for this time range.");

    var booking = new Booking {
        UserId = dto.UserId, ParkingSpotId = dto.ParkingSpotId,
        StartTime = dto.StartTime, EndTime = dto.EndTime,
        TotalPrice = dto.TotalPrice, Status = "Confirmed",
        SlotNumber = "A-" + new Random().Next(100, 999)
    };
    _context.Bookings.Add(booking);
    await _context.SaveChangesAsync();

    var user = await _context.Users.FindAsync(dto.UserId);
    if (user != null)
        await _emailService.SendEmailAsync(user.Email,
            "SmartParking Reservation Confirmed",
            $"<p>Slot {booking.SlotNumber} at {spot.Name} confirmed!</p>");

    return Ok(booking);
}

SmtpEmailService

public async Task SendEmailAsync(string toEmail, string subject, string body)
{
    var mailMessage = new MailMessage {
        From = new MailAddress(fromEmail),
        Subject = subject, Body = body, IsBodyHtml = true
    };
    mailMessage.To.Add(toEmail);
    using var smtpClient = new SmtpClient(host, port) {
        Credentials = new NetworkCredential(username, password),
        EnableSsl = true
    };
    await smtpClient.SendMailAsync(mailMessage);
}

Chapter 6: Limitations and Enhancement

6.1 Drawbacks and Limitations

  • Plain-text Password Storage: In the current implementation, passwords are stored as plain text in the PasswordHash field. A production system must use a cryptographic hash (e.g., BCrypt or Argon2).
  • No Payment Integration: The system calculates total prices but does not integrate with any payment gateway (Razorpay, Stripe, etc.). Users confirm bookings without online payment.
  • No Real-time Slot Availability: Spot availability is calculated based on bookings, but there is no WebSocket-based live update when another user takes a spot simultaneously.
  • Single Server Deployment: The application is designed for local deployment (localhost:8000). Cloud deployment (Azure, AWS, Render) with HTTPS, load balancing, and auto-scaling is not yet configured.
  • No Mobile Application: The frontend is web-only. Native iOS and Android apps are not yet developed.
  • Manual GPS Coordinates: Hosts must manually enter latitude/longitude. Integration with Google Maps Places API for auto-fill is not yet implemented.
  • No OTP Rate Limiting: The forgot-password OTP endpoint does not implement rate limiting, making it theoretically susceptible to spam requests.
  • No Push Notifications: Email is the only notification channel. SMS (Twilio) and web push notifications are not yet integrated.

6.2 Proposed Enhancements

  • BCrypt Password Hashing: Replace plain-text storage with BCrypt to meet industry security standards.
  • Razorpay / Stripe Payment Gateway: Integrate online payment collection at booking confirmation to enable a complete transactional flow.
  • WebSocket Live Updates: Use SignalR to push real-time parking spot availability updates to all connected clients simultaneously.
  • Google Maps API Integration: Auto-populate GPS coordinates and display parking spots on an interactive map with visual markers.
  • Mobile App (React Native / Flutter): Develop cross-platform mobile apps that consume the existing REST API backend.
  • OTP Rate Limiting & CAPTCHA: Add rate limiting (5 requests/hour per IP) to the forgot-password endpoint and integrate reCAPTCHA.
  • IoT Hardware Integration: Connect the system to physical barrier gates and sensors at parking entrances for automated vehicle detection and entry/exit logging.
  • Analytics Dashboard: Provide hosts and admins with revenue analytics, peak hour heatmaps, and occupancy trend charts.
  • Multi-language Support: Implement i18n/l10n for Hindi, Marathi, and other regional languages to serve a broader user base.
  • Cloud Deployment: Deploy on Azure App Service with Azure MySQL Flexible Server, HTTPS SSL certificate, and CI/CD via GitHub Actions.

6.3 Conclusions

The SmartParking Management System successfully demonstrates the design and implementation of a modern, full-stack web application that addresses a real-world urban problem. The project delivers a complete, working solution that covers the entire parking lifecycle — from spot discovery and reservation to cancellation and password management — all with automated email notifications at every step.

The system was built on a robust, industry-standard technology stack: ASP.NET Core 8.0 for the backend API, Entity Framework Core with MySQL for persistent storage, JWT-based authentication for secure sessions, and Gmail SMTP for transactional emails. The frontend, built with vanilla HTML, CSS, and JavaScript, provides a premium, responsive user experience without relying on heavy frameworks.

All key functional requirements were implemented and validated through rigorous black-box, white-box, and end-to-end Chrome browser tests. The automated test suite confirmed that all 11 test cases passed successfully, including a live E2E test that delivered four distinct emails to a real Gmail inbox, proving the system's production readiness.

This project provided invaluable hands-on experience in REST API design, relational database modeling, JWT security, SMTP email integration, and frontend-backend integration. The modular architecture ensures that the planned enhancements — payment gateways, IoT integration, and cloud deployment — can be layered on top of the existing codebase with minimal refactoring.

SmartParking is a solid foundation for a commercially viable parking management product that can scale to serve smart cities, malls, hospitals, and corporate campuses.
